<objective>
Fix two known gaps in AwsNovaProvider: (1) the audio transcript fallback uses hardcoded `_chunk_text()` params instead of the config-aware `chunk_text_with_strategy()`, and (2) video ingestion reads entire files into memory with no size guard, which will crash or trigger AWS payload rejection for files over 25MB.

Purpose: These are the two highest-risk production bugs for Nova - audio chunking silently ignores user's chunking config, and large video files will cause silent payload failures or OOM.
Output: Updated `AwsNovaProvider.build_audio_targets` and `AwsNovaProvider._embed_video` in `pinecone-multimodal-pipeline.py`.
</objective>

<task type="auto">
  <name>Task 2: Add video payload size guard to AwsNovaProvider</name>
  <files>pinecone-multimodal-pipeline.py</files>
  <action>
In `AwsNovaProvider._embed_video` (around line 1197), the method calls `file_path.read_bytes()` unconditionally. AWS Bedrock Nova has a 25MB request body limit. Large video files will cause boto3 to raise `ClientError` or silently produce empty embeddings.

**Step 1 — Add config field:**
In `PipelineConfig` dataclass, add:
```python
aws_nova_video_max_bytes: int  # default 20MB safety margin
```

In `PipelineConfig.from_env()`, add:
```python
aws_nova_video_max_bytes=int(_env("AWS_NOVA_VIDEO_MAX_BYTES", str(20 * 1024 * 1024))),
```

**Step 2 — Add guard in `_embed_video`:**
At the start of `AwsNovaProvider._embed_video`, before constructing the payload, add:
```python
file_size = file_path.stat().st_size
if file_size > self.config.aws_nova_video_max_bytes:
    print(
        f"[nova] Video {file_path.name} ({file_size // (1024*1024)}MB) exceeds "
        f"AWS_NOVA_VIDEO_MAX_BYTES ({self.config.aws_nova_video_max_bytes // (1024*1024)}MB). "
        "Skipping native embed — add video to manifest for metadata-only indexing."
    )
    return []
```

This makes `_embed_video` return `[]`, which causes `build_video_targets` to return `[]`, and `load_all` will skip the file (no crash, no silent failure).

**Step 3 — Add env var to `.env_sample`:**
Under the `AWS_NOVA_EXPECTED_DIMENSION` line, add:
```
AWS_NOVA_VIDEO_MAX_BYTES="20971520"  # 20MB default (Bedrock limit ~25MB, leave margin)
```
